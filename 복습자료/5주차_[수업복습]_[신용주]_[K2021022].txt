데이터 전처리는 알고리즘 만큼 중요하다
알고리즘은 데이터에 기반하기때문에 어떤 데이터를 입력으로 가지느냐에 따라 결과가 크게달라진다 
( garbage in, garbage out) 
- 사이킷런 ML 알고리즘은 결측값을 허용하지않음 (NaN, Null) 
- 결측값은 고정된 다른값으로 변환
- 결측값이 얼마안되면 피처의 평균값 , 대부분 결측값이라면 해당 피처 드랍
- 결측값이 일정 수준이상( 기준은 없음, 상황에 따라 다름) 업무상 로직에 맞춰 더 정밀한 값 대체[애매하다]
- 사이킷런 ML 알고리즘은 문자열 값을 입력으로 허용하지 않음 -> 인코딩하여 숫자로 변환하여야함
- 텍스트형 벡터는 피처 벡터화 하거나 불필요할경우 삭제 
- 식별자 피처는 로우를 식별하는 용도로 사용하여 예측에 중요한요소가 되지않고 알고리즘을 복잡하게 만들고 성능을 떨어트림 
- 데이터 인코딩의 방식은 레이블인코딩, 원핫인코딩  두가지
- 레이블 인코딩은 카테고리 피처를 코드형 숫자값으로 변환( 간단한 문자열을 숫자형 카테고리로 변환) 
- 숫자로변환되면 가중치가 다르게 부여되서 선형회귀에는 적용하지 말아야함 
- 트리계열 알고리즘에 반영하면 좋음 
- 원핫인코딩은 고유값엔 1 아닌값은 0 
- 서로다른값의 변수를 일정수준으로 맞추는 작업을 피처 스케일링이라고 함 대표적으로 표준화(standardization)와 정규화(normalization) 
- 표준화는 평균이 0이고 분산이 1인 가우시안 분포를 가진값으로 변환되는것을 의미
- 정규화는 서로 다른 피처의 크기를 통일하기위해 크기를 변환 
- 학습/테스트 데이터에 동일한 스케일링 정보 적용 
- 머신러닝은 데이터 가공/변환, 모델 학습/예측 , 그리고 평가의 프로세스로 진행 
-  정확도(Accuracy)는 실제 데이터에서 예측데이터가 얼마나 같은지를 판단하는 지표 
-  오차행렬(confusion matrix) 학습된 분류모델이 예측을 수행하면서 얼마나 헷갈리고있는지를 보여주는 지표
( 이진 분류의 예측 오류가 얼마인지, 어떤유형인지) 
- 정밀도는 예측을 Positive 로 한 대상 중에 실제값이 일치한 데이터의 비율 
재현율은 실제값이 positive 대상중 예측과 실제값이 일치한 비율 
- 모델에 따라서 중요한 지표가 달라짐( 암환자는 재현율이 중요) 
- 정밀도/재현율의 트레이드 오프 ( 한쪽을 높이면 한쪽이 떨어짐) 
- 지표를 높이기위한 숫자놀음이 될수도있다 
- F1 스코어는 정밀도와 재현율을 결합한 지표
